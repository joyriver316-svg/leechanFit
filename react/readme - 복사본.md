[과업지시서 V2] 헬스장 코치용 출석 관리 및 데이터 분석 시스템 구축
1. 프로젝트 개요
프로젝트명: 헬스장 코치 전용 출석 체크 및 운영 관리 대시보드

목적: 코치가 직접 고객의 출석을 기록하여 정확한 수업 참여 및 방문 데이터를 수집하고, 이를 기반으로 통계 분석 및 정산 데이터를 확보함.

주요 대상: 헬스장 코치 및 센터 운영자 (관리자용 웹/모바일 앱)

2. 주요 기능 요구사항
가. 코치용 출석 체크 시스템 (Manual Check-in)
고객 선택 기능: 전체 회원 명단에서 이름이나 연락처 검색을 통해 출석 대상 고객 선택.

시간 설정 기능: * 기본값은 '현재 시간'으로 자동 설정.

필요 시 코치가 과거 또는 특정 시간대를 수동으로 선택하여 소급 입력 가능.

수업 종류 선택: 일반 헬스 이용, PT(1:1), 그룹 수업 등 방문 성격을 선택하여 기록.

일괄 출석 처리: 여러 명의 고객을 선택하여 한 번에 동일한 시간으로 출석 처리하는 기능(그룹 수업용).

나. 관리자 대시보드 및 시각화 (Analysis)
시간대별 방문 고객 통계: 코치가 입력한 데이터를 바탕으로 시간대별 유동 인구 현황 그래프 제공.

날짜별 방문 일정 확인: 달력(Calendar) 형태의 UI를 통해 일자별 출석 인원 및 상세 명단 확인.

코치별 관리 통계: 각 코치가 담당하는 고객들의 출석률 및 수업 진행 횟수 통계.

미방문자 추적: 예약된 일정에 오지 않았거나, 일정 기간 방문 기록이 없는 고객 자동 추출.

다. 데이터 관리 및 엑셀 기능 (Reporting)
필터링 시스템: 특정 날짜 범위, 특정 코치, 특정 고객별로 출석 데이터를 자유롭게 조회.

엑셀/CSV 다운로드: * 고객별 통계 리포트: 선택한 기간 동안 특정 고객의 총 방문 횟수 및 일자 데이터 추출.

전체 출석부: 대시보드의 데이터를 정산 및 증빙용 엑셀 파일로 저장.

3. UI/UX 요구사항
현장 최적화 모바일 UI: 코치가 이동 중에도 한 손으로 빠르게 고객을 검색하고 시간을 선택할 수 있는 직관적인 인터페이스.

빠른 피드백: 출석 버튼 클릭 시 성공 여부를 즉시 알림(Toast 메시지 등)으로 표시.

달력 뷰(Calendar View): 날짜별 고객 방문 일정을 시각적으로 한눈에 파악할 수 있는 스케줄러 형태 지원.

4. 데이터 구조 (Schema) 수정안
Attendance Log: 코치 ID(기록 주체), 고객 ID, 기록된 방문 시간, 실제 입력 시간(Log), 수업 유형(일반/PT).

Coach Info: 코치 이름, 담당 고객 리스트, 권한 설정.

5. 향후 확장 고려 사항
푸시 알림: 코치가 출석 체크를 완료하면 해당 고객에게 카카오톡이나 앱 푸시로 "출석 완료" 및 "남은 횟수" 알림 발송.

예약 시스템 연동: 사전 예약된 명단을 불러와서 '출석' 버튼만 누르면 처리가 완료되는 프로세스 구현.
